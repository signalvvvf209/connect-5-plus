# Connect 5 Plus 設計書

| 項目 | 内容             |
|------|----------------|
| バージョン | セマンティックバージョニング |
| 言語 | Java 21        |
| パッケージ | `connect5plus` |
| 実行形式 | コンソールアプリケーション  |

---

## 1. 概要

Connect 5 Plus は、Connect 4（四目並べ）を拡張した 2人対戦型のボードゲームである。通常の **5 連** に加え、盤面上に **十字形 または X字形の 5 駒** でも勝利できる。

プレイヤーは列番号（x 座標）を指定して駒を落とし、先に勝利条件を満たしたプレイヤーが勝者となる。盤面が満杯で勝者がいない場合は引き分けとなる。

---

## 2. ゲームルール

### 2.1 基本ルール

| 項目 | 内容 |
|------|------|
| プレイヤー数 | 2 人 |
| 盤面サイズ | 9 × 9（デフォルト） |
| 駒の落下 | 指定した列の **最下段の空きマス** に駒が置かれる |
| 手番 | Player 1（●）→ Player 2（○）を交互に繰り返す |
| 終了条件 | 勝利 / 引き分け / プレイヤーによる中断（`e` 入力） |

### 2.2 勝利条件

盤面の駒が以下のいずれかを満たすと勝利する。

#### A. 5 連（直線）

同一プレイヤーの駒が、次のいずれかの方向に **5 個以上** 連続している。

- 縦
- 横
- 斜め（／、＼）

#### B. 十字, X字（5 駒）

同一プレイヤーの駒 5 つが、次のいずれかのパターンを形成している。

**B-1. 十字**

中心の駒から、上下左右の隣接マスすべてに同じ駒がある。

```
  ●
● ● ●
  ●
```
**B-1. X字**

中心の駒から、4 方向の斜め隣接マスすべてに同じ駒がある。

```
●   ●
  ●
●   ●
```

### 2.3 引き分け

全てのマスに駒が置かれ、勝者が決まらない場合。

### 2.4 勝利時の表示

勝利確定時、勝ち筋の マスは盤面コピー上でハイライト表示される。

| プレイヤー | 通常表示 | 勝利時ハイライト |
|-----------|---------|----------------|
| Player 1 | ● | ◆ |
| Player 2 | ○ | ◇ |

## 3. システム構成

| クラス               | 責務                         |
|-------------------|----------------------------|
| `Main`            | **エントリポイント**。UTF-8 設定、ゲームモード選択 |
| `Game`            | ゲーム進行（手番管理、入力、勝敗判定の統合、結果表示） |
| `AutoGame`        | 自動対戦（動作確認用）     |
| `SemiAutoGame`    | 半自動対戦（プレイヤーは1P） |
| `Board`           | 盤面状態、駒の落下、勝利判定、盤面表示、コピー    |
| `Token`           | 駒の種類（type）と表示文字            |
| `Position`        | 盤面上の座標（x, y）               |
| `BoardVector`     | 方向ベクトル（勝利判定の走査に使用）         |
| `ConsoleEncoding` | コンソール入出力の UTF-8 設定         |
| `Version`         | ビルド時生成されるバージョン定数           |

## 4. データモデル

### 4.1 座標系

```
縦方向: y
横方向: x
```

- `space[y][x]` で 2 次元配列に格納
- 駒は 下から 空きマスを探して配置（Connect Four 方式）

### 4.2 Token の type

| type | 意味 | 表示 |
|------|------|-----|
| 0 | 空（未使用） | （空白） |
| 1 | Player 1 | ● |
| 2 | Player 2 | ○ |
| 11 | Player 1 勝利ハイライト | ◆ |
| 12 | Player 2 勝利ハイライト | ◇ |

Player 番号（1, 2）をそのまま `Token` の type として使用する。勝利ハイライトは `10 + player` で表現する。

### 4.3 Game の状態

| フィールド | 初期値 | 意味 |
|-----------|--------|------|
| `player` | 1 | 現在の手番プレイヤー |
| `winner` | 0 | 0: 未決定, 1/2: 勝者, -1: 中断 |
| `winningPositions` | 空リスト | 勝利時の 5 マス座標 |
| `board` | 9×9 空盤面 | 盤面 |

### 4.4 Board の状態

| フィールド | 意味 |
|-----------|------|
| `boardSize` | 盤面の一辺の長さ |
| `space` | `Token[boardSize][boardSize]` の駒配置 |
| `tokenCount` | 盤面上の駒数（引き分け判定に使用） |

## 5. 拡張ポイント

| 拡張 | 方法 |
|------|------|
| 自動対戦 | `AutoGame`（`selectPosition()` をオーバーライド） |
| AI 対戦 | `Game` を継承し `selectPosition()` を実装 |
| 盤面サイズ変更 | `Board(int boardSize)` で生成（現状 `Game` は 9 固定） |
| プレイヤー数変更 | `playerMax` と `Token.typeToString` の拡張が必要 |

## 6. ビルド・実行

### 6.1 要件

- JDK 21
- Gradle（Wrapper 同梱）

### 6.2 コマンド

```bash
# ビルド
./gradlew build

# 通常プレイ
./gradlew run

# 半自動対戦
./gradlew run --args="-e"

# JAR 生成
./gradlew jar
```

### 6.3 バージョン管理

`build.gradle.kts` の `version` プロパティを変更すると、ビルド時に `build/generated/sources/version/java/connect5plus/Version.java` が自動生成される。
